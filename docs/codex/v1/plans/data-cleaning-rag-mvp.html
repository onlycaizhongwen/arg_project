数据清洗与 RAG 服务 MVP 版本

目标：先跑通一条可演示、可追溯、可扩展的最小 RAG 数据闭环；MVP 只做基础粗排与候选规模控制

5 个实施阶段

Step 1
工程与基础设施

  • 初始化 API 服务与 Worker 骨架
  • 准备数据库、Redis、MQ、对象存储、向量库
  • 定义统一配置与启动方式
  • 确认服务间连通性

Step 2
元数据与接口契约

  • 设计 data_source、document、document_version
  • 设计 cleaning_job、text_chunk、vector_record
  • 定义文件上传、任务查询、检索 API
  • 补充状态机和错误码

Step 3
文件接入与清洗

  • 上传文件保存到对象存储或本地目录
  • 创建清洗任务并投递 MQ
  • Worker 消费任务并执行解析
  • 输出清洗后的纯文本或结构化文本

Step 4
切分与向量入库

  • 按章节、段落和长度生成 chunk
  • 绑定文档、版本、页码、章节元数据
  • 调用 Embedding 适配层
  • 写入向量库并记录 vector_record

Step 5
语义检索闭环

  • 提供检索 API
  • 查询文本生成向量
  • 向量库召回 TopK
  • 基础粗排或分数融合控制候选规模
  • 暂不接 Cross-Encoder 精排
  • 返回片段、来源、分数和元数据

首期技术决策

后端技术栈 推荐 FastAPI 控制面 + Python Worker；统一 Python，API 与异步清洗解耦。
向量库 推荐 Qdrant 或 Milvus；小规模 MVP 可用 pgvector 快速验证。
消息队列 MVP 推荐 RabbitMQ；高吞吐流式场景再考虑 Kafka。
文件存储 推荐 MinIO；临时 MVP 可本地目录，但接口保留 object_uri。
模型服务 Embedding 使用通义/阿里云百炼，默认 text-embedding-v4，并保留适配层。

MVP 范围

必须做

  • 工程骨架和基础设施连通
  • 元数据表和核心状态机
  • 文件上传和任务查询
  • 至少 2 类文档格式解析
  • 文本切分、Embedding、向量入库
  • 语义检索和基础粗排/候选截断

暂不做

  • 数据库同步和第三方系统集成
  • 复杂 OCR 和音视频转写
  • 完整后台管理页面
  • 多轮问答编排
  • 领域模型微调
  • 业务干预：SimHash / MinHash 去重
  • 业务干预：MMR / DPP 打散
  • Cross-Encoder 精排

验收标准

可演示闭环

  • 上传样例文档后任务可从 UPLOADED 流转到 INDEXED
  • 检索问题可以召回目标文档片段
  • 结果包含文档 ID、chunk ID、来源、分数和元数据
  • 失败可定位到任务、文档版本和错误阶段
  • 候选规模可配置，MVP 不接精排模型，也不允许为后续精排留下大候选直传路径

最小 MVP 时序图

A. 文档上传与异步入库

sequenceDiagram
    autonumber
    actor U as 用户/业务系统
    participant API as FastAPI 控制面
    participant DB as PostgreSQL 元数据
    participant OBJ as MinIO 对象存储
    participant MQ as RabbitMQ 队列
    participant W as Python Worker
    participant V as Qdrant 向量库

    U->>API: 上传文件 POST /ingestions/files
    API->>OBJ: 保存原始文件
    OBJ-->>API: 返回 object_uri
    API->>DB: 创建文档记录与清洗任务(PENDING)
    API-->>MQ: 投递异步清洗任务(job_id)
    API-->>U: 返回 job_id / PENDING

    MQ-->>W: Worker 消费任务
    W->>DB: 更新 RUNNING / PARSING
    W->>OBJ: 读取原始文件
    W->>W: 解析、清洗、切分
    W->>W: 生成 Embedding
    W->>V: 写入 chunk 向量
    W->>DB: 写入 chunk/vector_record，更新 SUCCESS

B. 语义检索与基础粗排

sequenceDiagram
    autonumber
    actor U as 用户/业务系统
    participant API as FastAPI 控制面
    participant E as Embedding 适配层
    participant V as Qdrant 向量库
    participant DB as PostgreSQL 元数据

    U->>API: 发起检索 POST /rag/search
    API->>E: 生成 query embedding
    E-->>API: 返回 query vector
    API->>V: 向量召回 recall_size 候选
    V-->>API: 返回 chunk_id + score
    API->>API: 基础粗排 / 候选截断(pre_rank/top_k)
    API->>DB: 查询 chunk 文本、来源、元数据
    DB-->>API: 返回可追溯信息
    API-->>U: 返回片段、来源、分数、metadata
Mermaid 约定：实线表示同步调用，虚线返回表示响应或异步投递结果。MVP 只做语义召回和基础粗排/候选截断，不接 Cross-Encoder 精排，也不做 SimHash/MinHash 去重和 MMR/DPP 打散。
