# 客服知识库接入说明

数据清洗与 RAG 服务用于把客服知识库文档接入统一检索链路。业务系统通过 FastAPI 上传文件，原始文件进入 MinIO，文档、版本、清洗任务、文本片段和向量记录写入 PostgreSQL。

清洗任务通过 RabbitMQ 投递给 Python Worker。Worker 会解析文件、清理空白字符、按段落切块，并调用当前配置的 Embedding provider。内网演示环境推荐使用 local_bge 和 bge-m3，向量维度为 1024；生产环境可以切换到通义 DashScope text-embedding-v4。

向量写入 Qdrant 时会携带 tenant_id、knowledge_base_id、document_id、document_version_id 和 permission_tags。检索接口会使用 knowledge_base_id 和权限标签做过滤，确保不同客户、不同业务线的数据不会串库。
