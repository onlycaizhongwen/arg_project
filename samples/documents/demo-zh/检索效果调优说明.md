# 检索效果调优说明

检索链路采用召回、粗排、业务干预和精排的漏斗思路。MVP 阶段已具备向量召回、关键词召回、候选合并、候选截断、去重、打散和可选 rerank 的基础能力。

如果召回结果太少，可以提高 recall_size；如果响应时间变慢，可以降低 pre_rank_size 或 top_k。业务干预阶段可以设置 max_chunks_per_document，避免单个文档占满结果列表。

如果接入本地 BGE rerank，需要关注 rerank_size 和 RERANK_TIMEOUT_SECONDS。rerank 服务不可用时，系统会降级返回粗排结果，并记录 RERANK_DEGRADED 诊断事件。

评测报告需要同时观察 Hit Rate、MRR、Recall@K、P50/P95/P99 延迟和 rerank 降级次数。PoC 推荐先使用 local_bge + mock rerank 做稳定演示，再用 DashScope 或本地 rerank 做效果对比。
