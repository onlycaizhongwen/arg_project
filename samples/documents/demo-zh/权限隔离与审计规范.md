# 权限隔离与审计规范

RAG 检索必须遵守租户、知识库和权限标签三层隔离。tenant_id 表示租户，knowledge_base_id 表示知识库，permission_tags 表示文档可见范围。

上传文档时可以通过 X-Tenant-Id、X-Actor-Id、X-Request-Source 和 X-Permission-Tags 注入请求上下文。Header 的优先级高于 query 或 body 参数，适合接入网关、控制台或第三方业务系统。

检索请求中的 permission_context 会与文档 permission_tags 取交集。只有存在交集的片段才会返回。例如文档标签为 finance 和 internal，用户上下文只有 public 时不能看到该文档。

审计事件需要记录 actor_id、request_source、operation、document_id、job_id 和 metadata。重要操作包括文档更新、删除、重建、批量重建、失败任务重试和滞留锁释放。
