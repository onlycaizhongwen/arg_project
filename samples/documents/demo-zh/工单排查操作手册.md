# 工单排查操作手册

当用户反馈“上传成功但搜索不到内容”时，值班人员应先查看 cleaning job 状态。如果 job 仍为 PENDING 或 RUNNING，需要等待 Worker 继续处理；如果 job 为 FAILED，需要查看 error_message 和 Worker 日志。

排查步骤：

1. 调用 `/api/v1/jobs/{job_id}` 查看任务状态。
2. 调用 `/api/v1/diagnostics/overview` 查看 RabbitMQ ready 消息数、consumer 数、失败率和 document 操作锁。
3. 如果 RabbitMQ consumer_count 等于 0，优先重启 Worker。
4. 如果 document 操作锁超过阈值且关联 job 已失败，可以调用 locks release 接口安全释放滞留锁。
5. 释放后对文档发起 rebuild，确认文本片段和 Qdrant 向量重新生成。

该流程要求所有人工操作都写入审计事件，便于对客复盘。
